2. XAC THUC VA PHAN QUYEN
-------------------------

Endpoint co `requireAuth` can header:

  Authorization: Bearer <accessToken>

Access token phai:
- La JWT access token hop le va chua het han.
- Co user tuong ung trong DB.
- User co status `active`.

Neu thieu token:
- HTTP 401
- Message: `Access token is required`

Neu token sai hoac het han:
- HTTP 401
- Message: `Access token is invalid or expired`

Admin route nam duoi `/api/admin/...` can them role `admin`.
Neu access token hop le nhung role khong phai admin:
- HTTP 403
- Message: `Admin access is required`

Token:
- Login tra ve accessToken va refreshToken.
- Refresh endpoint tra ve accessToken va refreshToken moi; refresh token cu bi rotate.
- Refresh token duoc luu DB dang hash va co TTL.
- TTL mac dinh tu code: access token 15 phut, refresh token 7 ngay neu cac
  bien JWT_ACCESS_TTL_SECONDS va JWT_REFRESH_TTL_SECONDS khong duoc dat.


3. LOI CHUNG
------------

Thanh cong khong boc trong envelope `success`. Moi endpoint tra object response
rieng cua endpoint do.

Loi qua global error handler co shape:

  {
    "success": false,
    "error": {
      "statusCode": 400,
      "code": "VALIDATION_ERROR",
      "message": "Mo ta loi",
      "details": "co the co hoac khong"
    }
  }

Code loi thuong gap:
- VALIDATION_ERROR: body, query hoac path param khong hop le.
- BAD_REQUEST: loi request 400 khac.
- UNAUTHORIZED: 401.
- FORBIDDEN: 403.
- NOT_FOUND: 404.
- CONFLICT: 409 tu service.
- DUPLICATE_KEY: duplicate key MongoDB chua duoc service chuyen doi.
- CAST_ERROR: Mongoose cast loi.
- INTERNAL_SERVER_ERROR: loi 5xx; message tra ve client la
  `Internal server error`.

Route khong ton tai tra 404 theo cung error shape, vi du message:
`Route GET /duong-dan-sai not found`.

Path/body productId, categoryId va orderId o cac endpoint co validation phai la
MongoDB ObjectId hop le.

6. AUTH API
-----------

6.1 Dang ky

POST /api/auth/register

Access:
- Public.

Body:

  {
    "email": "user@example.com",
    "password": "mat-khau-it-nhat-8-ky-tu",
    "name": "Nguyen Van A"
  }

Validation va hanh vi:
- `email`, `password`, `name` bat buoc la string khong rong.
- `password` can toi thieu 8 ky tu.
- User model kiem tra dinh dang email.
- Email duoc trim va lowercase truoc khi luu.
- Name duoc trim truoc khi luu.
- Role moi mac dinh `customer`; status moi mac dinh `active`.

Success:
- HTTP 201

  {
    "user": <Auth user>
  }

Loi nghiep vu dang thay trong code:
- 409 neu email da dang ky.

6.2 Dang nhap

POST /api/auth/login

Access:
- Public.

Body:

  {
    "email": "user@example.com",
    "password": "mat-khau"
  }

Validation va hanh vi:
- `email`, `password` bat buoc la string khong rong.
- Email duoc trim va lowercase de tim user.

Success:
- HTTP 200

  {
    "user": <Auth user>,
    "accessToken": "string",
    "refreshToken": "string"
  }

Loi nghiep vu dang thay trong code:
- 401 neu email hoac password sai.
- 403 neu account bi banned.

6.3 Lam moi access token

POST /api/auth/refresh-token

Access:
- Public; refresh token di trong body.

Body:

  {
    "refreshToken": "string"
  }

Success:
- HTTP 200

  {
    "accessToken": "string",
    "refreshToken": "string"
  }

Loi nghiep vu dang thay trong code:
- 401 neu refresh token sai, het han, khong con record hop le trong DB hoac
  user khong con dung duoc.

6.4 Dang xuat session refresh token

POST /api/auth/logout

Access:
- Public; refresh token di trong body.

Body:

  {
    "refreshToken": "string"
  }

Success:
- HTTP 200

  {
    "message": "Logged out"
  }

Ghi chu:
- Service xoa record co hash trung voi refreshToken nhan duoc.
- Endpoint khong can Authorization header va khong verify JWT refresh token
  truoc khi delete record.

7.2 Danh sach eyeglasses

GET /api/eyeglasses

Access:
- Public.

Query optional:
- `offset`: pagination offset. Mac dinh 0.
- `limit`: so record. Mac dinh 12; gioi han tren 100.
- `frameType`: `Acetate` hoac `Metal`.
- `collectionSlug`: slug collection; input duoc trim, lowercase va chuan hoa
  khoang trang/underscore thanh dau `-`.
- `gender`: filter chi nhan male/men hoac female/women, khong phan biet hoa
  thuong. Filter database duoc doi thanh `Male` hoac `Female`.
- `sale`: `true`, `1`, `yes`, `false`, `0`, `no`, khong phan biet hoa
  thuong. Mac dinh false. Khi true chi lay san pham co salePercent > 0.

Success:
- HTTP 200

  {
    "records": [<Eyeglasses product list item>],
    "total": 0
  }

Ghi chu:
- Query san pham chi lay `isActive: true`.
- Neu collectionSlug khong ton tai thi tra `{ "records": [], "total": 0 }`.
- `total` la tong record match filter truoc pagination.

- `gender`: male/men hoac female/women, khong phan biet hoa thuong.

10. ORDER API
-------------

10.1 Checkout cart

POST /api/orders/checkout

Access:
- Bearer access token.

Body:

  {
    "shippingAddress": {
      "recipientName": "Nguyen Van A",
      "phone": "0900000000",
      "line1": "123 Duong A",
      "line2": "optional",
      "ward": "optional",
      "district": "optional",
      "city": "Ho Chi Minh",
      "province": "optional",
      "postalCode": "optional",
      "country": "Vietnam"
    },
    "paymentMethod": "cash_on_delivery"
  }

Validation:
- `shippingAddress` bat buoc.
- Address field bat buoc: `recipientName`, `phone`, `line1`, `city`.
- Address field optional: `line2`, `ward`, `district`, `province`,
  `postalCode`, `country`.
- Neu khong gui country, middleware dat country mac dinh `Vietnam`.
- Moi field address duoc gui phai la string khong rong.
- `paymentMethod` optional; gia tri hop le:
  `cash_on_delivery`, `bank_transfer`, `card`, `e_wallet`.

Hanh vi:
- Checkout chay trong MongoDB transaction.
- Cart phai co item.
- Moi product/variant trong cart phai con ton tai, active, in_stock va du stock.
- Backend tru stock variant, tao order, tinh lineTotal/subtotal/total tu gia
  variant hien tai, sau do clear cart.
- Order moi mac dinh status `pending`, paymentStatus `unpaid`, currency `VND`.

Success:
- HTTP 201 `<Order>`

Loi nghiep vu dang thay trong code:
- 400 neu cart rong.
- 409 neu product trong cart khong con ton tai.
- 409 neu product/variant khong con ban duoc hoac khong du stock.
- 409 neu checkout xung dot voi stock update khac; client nen retry.

10.2 Admin cap nhat order status

PATCH /api/admin/orders/:orderId/status

Access:
- Bearer access token cua admin.

Path:
- `orderId`: MongoDB ObjectId.

Body:

  {
    "status": "processing"
  }

Status hop le:
- `pending`
- `confirmed`
- `processing`
- `shipped`
- `delivered`
- `cancelled`

Transition duoc service cho phep:
- pending -> confirmed | processing | shipped | delivered | cancelled
- confirmed -> processing | shipped | delivered | cancelled
- processing -> shipped | delivered | cancelled
- shipped -> delivered | cancelled
- delivered -> khong chuyen sang status khac
- cancelled -> khong chuyen sang status khac
- Gui lai dung status hien tai duoc chap nhan va tra order hien tai.

Success:
- HTTP 200 `<Order>`

Loi nghiep vu dang thay trong code:
- 404 neu order khong ton tai.
- 409 neu transition status khong hop le.

13. API BO SUNG THEO TAI LIEU IE213
-----------------------------------

Ngoai cac endpoint o cac muc tren, backend co cac nhom API sau de phu hop voi
tai lieu IE213:
- Auth: GET /api/auth/me, POST /api/auth/forgot-password,
  POST /api/auth/reset-password.
- User: /api/users/profile, /api/users/change-password,
  /api/users/addresses va DELETE /api/users/me.
- Category alias: /api/categories, bao gom GET theo slug va admin mutation
  POST/PUT/DELETE.
- Cart alias theo SKU: GET/POST/DELETE /api/cart va PUT/DELETE /api/cart/:sku.
- Order alias: POST/GET /api/orders, GET /api/orders/:orderId va
  PUT /api/orders/:orderId/cancel.
- Review interaction: POST /api/reviews/:reviewId/like,
  PUT /api/reviews/:reviewId va DELETE /api/reviews/:reviewId.
- Inventory: GET /api/inventory/check cho user; admin doc/cap nhat inventory
  qua /api/admin/inventory.
- Admin: doc user, doc order, update order status, xoa review va stats qua
  /api/admin/users, /api/admin/orders, /api/admin/reviews va /api/admin/stats.

Ghi chu tuong thich:
- Inventory hien duoc suy ra tu `variants.stock` cua Eyeglasses/Sunglasses.
  Backend chua co reservation collection, nen `reserved` luon la 0.
- Forgot-password tao reset token trong DB. Khi chua co mail provider, token
  duoc log ra console o moi truong khong phai production de test reset flow.
- Product API `/api/products` khong duoc them theo yeu cau.
